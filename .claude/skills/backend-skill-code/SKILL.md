## 强制约束规则
- 所有接口强制遵循RESful风格
- 所有接口必须添加 @Valid 参数校验，禁止裸参数接收
- 数据库操作优先使用MyBatis-Plus Lambda语法，禁止硬编码SQL
- 所有分页接口必须使用MyBatis-Plus分页插件，禁止手写分
- 所有接口统一返回 Result<T>，禁止直接返回实体、List、Map
- 强制空指针、集合空判断，杜绝NPE风险
- 新增业务必须编写JUnit5单元测试，覆盖正常、异常、边界场景
- 所有类、核心方法、复杂逻辑必须添加中文注释
- 严格单一职责，单方法代码不超过80行，禁止超大类、万能方法
- 禁止魔法值，所有固定参数统一维护在常量类
- 业务事务必须配置 @Transactional，严格遵循事务生效规则 页
- 事务方法必须为public，禁止内部调用、非事务方法调用导致事务失效
- 禁止在Controller写业务逻辑、数据库操作
- 异常统一抛出业务异常，禁止直接返回错误字符串
- 复杂业务推荐使用设计模式，禁止记流水账方式写业务代码

## 统一命名规范
- 类名：大驼峰，望文知意
- 方法/变量：小驼峰，动词+名词
- 常量：全大写，下划线分隔
- 数据表、接口路径：小写下划线分隔
- DTO后缀：xxxDTO（入参）
- VO后缀：xxxVO（出参）

## AI开发强制输出规则
1. 禁止生成冗余import、废弃代码、无效注释、空方法
2. 严格贴合现有项目结构，不随意新建包、修改架构
3. 禁止私自新增、修改、删除Maven/Gradle依赖，依赖变更必须人工审核
4. 代码生成后必须保证编译通过、无语法错误
5. 新增文件必须补齐标准头部注释、功能说明
6. 严格遵循团队统一代码风格、注释规范、异常规范
7. 修复Bug、重构代码不得改动原有正常业务逻辑
